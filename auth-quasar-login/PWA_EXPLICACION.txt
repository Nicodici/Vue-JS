PWA (Progressive Web App): explicación completa
=================================================

1) ¿Qué es una PWA?
-------------------
Una PWA (Progressive Web App o Aplicación Web Progresiva) es una aplicación web que utiliza tecnologías modernas del navegador para comportarse casi como una app nativa.

En otras palabras, una PWA se abre desde el navegador, pero puede:
- instalarse en el dispositivo,
- funcionar sin conexión (o con conexión limitada),
- enviar notificaciones push,
- cargarse rápido y ofrecer una experiencia de uso fluida.

Una PWA combina lo mejor de:
- una web (accesible por URL, sin pasar por tienda obligatoriamente),
- y una app móvil/escritorio (instalable, rápida, con acceso a ciertas capacidades del sistema).


2) ¿Para qué sirve una PWA?
---------------------------
Sirve para ofrecer una experiencia de aplicación moderna sin tener que desarrollar, mantener y publicar múltiples apps nativas para cada plataforma.

Usos comunes:
- E-commerce: catálogos y carrito accesibles incluso con mala red.
- Educación: contenidos que se consultan offline.
- Productividad: tareas, notas, paneles internos.
- Medios/noticias: carga rápida y notificaciones.
- Negocios locales: menús, reservas, promociones.

Objetivo principal:
- Mejorar rendimiento, accesibilidad y retención de usuarios.


3) Ventajas de una PWA
----------------------
1. Instalación sencilla
   - El usuario puede “Agregar a pantalla de inicio” sin pasar por una tienda.

2. Funciona offline o en redes inestables
   - Gracias al Service Worker y caché de recursos.

3. Mejor rendimiento
   - Recursos estáticos en caché, menor tiempo de carga percibido.

4. Menor costo de desarrollo y mantenimiento
   - Una sola base web para múltiples plataformas.

5. Actualizaciones inmediatas
   - Se actualiza en el servidor; el usuario recibe cambios sin instalaciones pesadas.

6. Mejor experiencia móvil
   - Diseño responsive, icono propio, apertura en pantalla completa (según configuración).

7. Seguridad
   - Requiere HTTPS (excepto localhost en desarrollo).

8. Posibilidad de notificaciones push
   - Útil para reenganche y comunicación con usuarios.


4) ¿Cómo se instala una PWA?
----------------------------
Desde el punto de vista del usuario final:

En Android (Chrome/Edge):
- Entrar al sitio.
- Esperar sugerencia "Instalar app" o abrir menú del navegador.
- Pulsar "Instalar" o "Agregar a pantalla de inicio".

En escritorio (Chrome/Edge):
- Entrar al sitio.
- Pulsar el icono de instalación en la barra de direcciones (si aparece).
- Confirmar instalación.

En iOS (Safari):
- Abrir el sitio en Safari.
- Compartir -> "Añadir a pantalla de inicio".

Nota:
- En algunos navegadores/plataformas la instalación depende de que la app cumpla ciertos criterios (manifest válido, HTTPS, service worker activo, etc.).


5) ¿Cómo se configura una PWA? (para desarrollo)
-------------------------------------------------
Para convertir una web en PWA, se configuran estos elementos clave:

A) Web App Manifest (manifest.webmanifest o manifest.json)
-----------------------------------------------------------
Es un archivo JSON con metadatos de la app.

Campos habituales:
- name: nombre completo
- short_name: nombre corto
- start_url: URL inicial
- display: standalone/fullscreen/browser
- background_color
- theme_color
- icons: diferentes tamaños y formatos

Ejemplo mínimo:
{
  "name": "Mi App PWA",
  "short_name": "MiPWA",
  "start_url": "/",
  "display": "standalone",
  "background_color": "#ffffff",
  "theme_color": "#1976d2",
  "icons": [
    {
      "src": "/icons/icon-192.png",
      "sizes": "192x192",
      "type": "image/png"
    },
    {
      "src": "/icons/icon-512.png",
      "sizes": "512x512",
      "type": "image/png"
    }
  ]
}


B) Service Worker
-----------------
Script que corre en segundo plano y permite:
- cachear recursos,
- interceptar peticiones,
- estrategia offline,
- push notifications (si aplica).

Pasos generales:
1. Crear el archivo del service worker.
2. Registrar el service worker desde tu app principal (main.js, etc.).
3. Definir estrategia de caché (cache-first, network-first, stale-while-revalidate, etc.).

Ejemplo de registro (simplificado):
if ('serviceWorker' in navigator) {
  window.addEventListener('load', () => {
    navigator.serviceWorker.register('/service-worker.js')
      .then(() => console.log('Service Worker registrado'))
      .catch((err) => console.error('Error SW:', err));
  });
}


C) HTTPS
--------
- Obligatorio en producción para funcionalidades PWA.
- En local suele permitirse en localhost para pruebas.


D) Iconos y metadatos
---------------------
- Añadir iconos en tamaños correctos (192, 512, etc.).
- Configurar color de tema y pantalla de inicio.


E) Estrategia de caché
----------------------
Definir qué se cachea y cuándo se actualiza:
- App shell (HTML base, CSS, JS críticos).
- Recursos estáticos.
- Respuestas API (si aplica, con criterios claros).


6) Flujo recomendado de implementación
--------------------------------------
1. Aplicación web funcional y responsive.
2. Agregar y enlazar manifest.
3. Crear/registrar service worker.
4. Definir estrategias de caché.
5. Probar offline y rendimiento.
6. Validar criterios instalables.
7. Desplegar con HTTPS.


7) Herramientas útiles
----------------------
- Lighthouse (Chrome DevTools): auditoría de PWA, performance y buenas prácticas.
- Workbox: librería para simplificar service worker y estrategias de caché.
- Frameworks con soporte PWA (Vue, React, Angular, Quasar, Vite plugins).


8) Limitaciones y consideraciones
---------------------------------
- Algunas APIs avanzadas dependen del navegador/plataforma.
- En iOS hay restricciones frente a Android/desktop en ciertas capacidades.
- Si la caché se diseña mal, puede mostrar contenido desactualizado.
- Requiere planificación de versionado y actualización del service worker.


9) Resumen rápido
-----------------
Una PWA es una web mejorada que puede instalarse, funcionar offline y ofrecer una experiencia similar a una app nativa.

Es ideal cuando quieres:
- buena experiencia multiplataforma,
- menor costo de desarrollo,
- rendimiento alto,
- y despliegue web rápido con posibilidad de instalación.


10) Ejemplo práctico: convertir tu app Quasar en PWA
----------------------------------------------------
Este ejemplo está pensado para un proyecto Quasar CLI moderno.

Paso 1: Agregar el modo PWA al proyecto
----------------------------------------
Desde la raíz de tu proyecto Quasar, ejecuta:

quasar mode add pwa

Esto agrega la estructura necesaria para PWA (service worker/archivos relacionados según configuración).


Paso 2: Configurar `quasar.config.js`
-------------------------------------
En el archivo `quasar.config.js`, asegúrate de tener el bloque `pwa` con manifest e íconos.

Ejemplo base:

// quasar.config.js
import { configure } from 'quasar/wrappers'

export default configure(function () {
   return {
      // ...otras opciones

      pwa: {
         workboxMode: 'GenerateSW',
         injectPwaMetaTags: true,
         swFilename: 'sw.js',
         manifestFilename: 'manifest.json',
         useCredentialsForManifestTag: false,

         manifest: {
            name: 'Auth Quasar Login',
            short_name: 'AuthLogin',
            description: 'Aplicación de autenticación con Quasar en modo PWA',
            display: 'standalone',
            orientation: 'portrait',
            background_color: '#ffffff',
            theme_color: '#1976d2',
            icons: [
               {
                  src: 'icons/icon-128x128.png',
                  sizes: '128x128',
                  type: 'image/png'
               },
               {
                  src: 'icons/icon-192x192.png',
                  sizes: '192x192',
                  type: 'image/png'
               },
               {
                  src: 'icons/icon-256x256.png',
                  sizes: '256x256',
                  type: 'image/png'
               },
               {
                  src: 'icons/icon-384x384.png',
                  sizes: '384x384',
                  type: 'image/png'
               },
               {
                  src: 'icons/icon-512x512.png',
                  sizes: '512x512',
                  type: 'image/png'
               }
            ]
         }
      }
   }
})

Nota:
- Verifica que esos íconos existan en la carpeta `public/icons` (o la ruta que uses).


Paso 3: Probar en desarrollo como PWA
-------------------------------------
Ejecuta:

quasar dev -m pwa

Luego abre DevTools -> Application y revisa:
- Manifest cargado,
- Service Worker registrado,
- Opciones de instalación disponibles.


Paso 4: Compilar para producción
--------------------------------
Ejecuta:

quasar build -m pwa

El resultado se genera en `dist/pwa`.


Paso 5: Desplegar con HTTPS
---------------------------
Publica `dist/pwa` en un hosting con HTTPS (Netlify, Vercel, Firebase Hosting, etc.).

Sin HTTPS en producción, muchas funciones PWA no estarán disponibles.


Paso 6 (opcional): estrategia de caché avanzada
------------------------------------------------
Si necesitas más control offline (APIs, rutas específicas), cambia:
- `workboxMode: 'GenerateSW'` -> `workboxMode: 'InjectManifest'`

y define lógica personalizada en tu service worker.


Checklist rápida para tu Quasar PWA
-----------------------------------
- `quasar mode add pwa` ejecutado.
- `pwa.manifest` bien configurado.
- Íconos reales en tamaños requeridos.
- `quasar dev -m pwa` sin errores.
- `quasar build -m pwa` generado.
- Deploy final en HTTPS.
